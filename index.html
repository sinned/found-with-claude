/new-startup
New Startup Onboarding
Creates work/startup-N/ with all skill folders, sets it as the active workspace, and asks 4 questions to capture your founder context.
InYour answers to 4 onboarding questions
OutStartup workspace + founder-context.md + suggested first skill
/loop
Autonomous Founder Loop
Chains all skills in sequence. Runs autonomously until it hits a phase that needs real-world founder input — then stops, saves the block, and waits. Drop the input and run /loop again to resume.
Inloop-state.md (current position) + any new files you've dropped
OutAll skill outputs up to next block + updated loop-state.md
Step 1 Find a Problem — Is this pain real and underserved?
/founder-opportunity-map
Opportunity Map
Map and rank potential startup opportunities from your hunches, domain knowledge, and observations. Produces a scored opportunity memo.
InBackground, domain expertise, messy notes
OutRanked memo: 3–6 areas scored on pain, underservice, and founder fit
Step 2 Customer Discovery — Who has it and what do they say?
/customer-discovery-synthesizer
Discovery Synthesizer
Synthesize raw interview notes and transcripts into structured insight — patterns, jobs-to-be-done, quotes, and open questions. Works from 2+ interviews.
InRaw interview notes, transcripts, voice memo summaries
OutPatterns, JTBD, riskiest assumption, open questions
Step 3 Customer Simulation — What would real users say about this?
/synthetic-customer-panel
Synthetic Customer Panel
Simulate 4–6 customer archetype reactions to your concept before you build anything. A thinking tool for stress-testing ideas. Not a research substitute.
InCustomer synthesis + concept to test
OutArchetype reactions, cross-panel analysis, hypotheses to test
Step 4 Product Strategy — What do we build and why?
/product-strategy-brief
Product Strategy Brief
Synthesize all discovery work into a one-page build decision: target customer, core problem, solution hypothesis, key bets, and what's explicitly out of scope.
InOpportunity memo + customer synthesis + panel output
OutOne-page brief: target, problem, hypothesis, bets, out-of-scope
Step 5 Prototype — Can we build a working version fast?
/prototype-scope
Prototype Scope
Define exactly what to build, what success looks like, and how to hand off to the build environment. Keeps scope to one job.
InProduct strategy brief
OutScope doc, success criteria, readiness checklist, build handoff
prototype-with-claude ↗
Build the Prototype
The build environment for AI feature prototyping. Write a SKILL.md, run it in Claude Code, measure with evals, improve automatically.
InPrototype scope doc
OutWorking AI prototype, eval report
/prototype-review
Prototype Review
Evaluate the built prototype against the original strategy. Produces an honest verdict with next steps: Proven / Disproven / Inconclusive / Blocked.
InPrototype scope + product brief + what was actually built
OutHonest verdict + next steps
Step 6 First Users — Who are the first 10 users and how do we reach them?
/first-users-outreach
First Users Outreach
Plan and write outreach to get the first 10 users — warm, referral, and cold sequences with a feedback capture template built in.
InProduct brief + prototype description
OutOutreach sequence + onboarding ask + feedback capture guide
Step 7 PMF Sprint — Are we seeing the signals that matter?
/pmf-signal-review
PMF Signal Review
Analyze product-market fit signals from early user data, feedback, and behavior. Returns a clear verdict and concrete next steps. Needs 10+ users.
InUser data, feedback, retention signals, qualitative observations
OutSignal memo with verdict: Strong / Mixed / Weak
Step 8 Growth — What lever do we pull next?
/growth-experiment-backlog
Growth Experiment Backlog
Generate a prioritized backlog of growth experiments based on PMF signals and your current bottleneck. Each experiment includes a hypothesis, metric, and effort estimate.
InPMF signal memo + current growth bottleneck
Out3 experiments to run this week + full backlog with effort estimates
Step 9 Hiring — Who is the right first hire?
/founder-hiring-scorecard
Hiring Scorecard
Create a structured hiring scorecard for an early-stage role — must-haves, nice-to-haves, red flags, and interview questions calibrated to your stage.
InRole description + team context + what this hire must unlock
OutScorecard with criteria, red flags, interview questions
Step 10 Fundraising — What is the story investors need to hear?
/fundraising-narrative
Fundraising Narrative
Build the investor story from real traction data. Produces an investor memo and objection map. Requires real traction — a narrative without traction is a hypothesis presentation.
InProduct brief + PMF signal review + traction data
OutInvestor memo + objection map
Step 11 Demo Day — Can we land this in 2 minutes on stage?
/demo-day-script
Demo Day Script
Write a word-for-word 2-minute Demo Day pitch script: hook, problem, solution, traction, ask. Includes a timing guide and danger zones to avoid.
InFundraising narrative + live demo flow
OutWord-for-word script + timing guide + danger zones
Ongoing Review — What did we learn this week?
/weekly-founder-review
Weekly Founder Review
The meta-skill that ties everything together. Run every Friday or Monday. Forces three questions: what was learned, what was done, and what is the single most important next step.
InWins, blocks, conversations, data, personal state
OutStructured review + clear decision for next week
Get started

Up and running in 4 steps.

No API keys. No setup. No boilerplate. Just Claude Code and your raw founder material.

1
Clone the repo
Skills live in .claude/skills/ and are auto-detected by Claude Code when you open the project.
2
Open Claude Code
Run claude from the repo root. Skills load automatically from the project directory.
3
Run /new-startup
Creates your startup workspace in work/startup-1/, captures your context, and suggests the right first skill.
4
Run /loop
Chains all skills autonomously. Runs what it can, stops with exact instructions when it needs you, resumes where it left off when you drop the input.
