# Investor update format is stable for months or years — making it a predictable automation target

Both early interviewees reported using the same update format for an extended period,
driven by investor expectation. The format becoming a known quantity is what makes
automation tractable: same sources, same fields, same structure every cycle.

**Evidence:** 2/2 interviews confirmed this. Quote: "I've been using the same format
for 18 months. My investors actually get annoyed if I change it." The template
stability is enforced from the investor side, not just the founder's preference.

**Implication for product:** Automation becomes feasible precisely because the format
is stable. The product can let the founder define the template once and gather data
for it on every cycle. Do not try to infer structure from scratch each time — persist
the template configuration and reuse it.

**Open question:** How variable are templates across founders? If templates are highly
idiosyncratic, the product needs flexible template configuration. If most founders
use similar structures (MRR, burn, key hires, product highlight, ask), the product
can offer a starting template.
