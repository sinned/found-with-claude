# Seed-stage founders spend 90-120 minutes gathering data before a 30-minute investor update

Every recurring investor update requires pulling the same metrics from 4-6 sources
(Stripe, Mercury, Mixpanel, Linear, Salesforce, etc.) before writing can begin.
The gathering is purely mechanical — same sources, same fields, every cycle — yet
there is no trusted way to automate it because founders fear sending wrong numbers
to investors.

**Who has it:** Seed-stage founders sending weekly or bi-weekly investor updates.
Strongest signal from solo or near-solo founders without dedicated ops or finance
help. Also present in Series A founders with small teams.

**Severity:** 90-120 minutes of mechanical work per update cycle. If weekly, this
is ~6-8 hours per month of low-value work. Professional risk if a number is wrong
(investors notice).

**Current workarounds:** Fully manual: open each source separately, copy numbers,
paste into Google Doc or slide deck, manually calculate derived metrics (e.g., burn
rate), double-check every number before sending. Previous automation attempts
(Zapier, custom scripts) failed because errors in automated numbers were not caught
before sending.

**Evidence:** 2/2 initial interviews, plus direct founder experience at two companies.
Opportunity score: 14/15 (Pain: 5, Underservice: 4, Founder fit: 5).
