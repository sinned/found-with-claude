# Ag software gets abandoned when it requires manual data entry or only produces "pretty maps"; only load-bearing weekly-habit tools survive

Tools that demand the farmer/agronomist type in records, or that just visualize what the user
already knows, get set up once and then quietly abandoned. The products that stick are opened
weekly because they are load-bearing for a recurring decision.

**Evidence:** 3/5 simulated interviews. Dale churned FieldView: "Pretty maps don't pay the
loan." Tyler: "Every farmer app dies on data entry... the ones that stick are the ones I open
every single week because they're load-bearing." Greg: a tool "can't fight our sales
system — it's got to feed the order."
**Implication for product:** Auto-ingest as-applied/spray data (no manual entry); deliver a
specific recurring action (e.g., warn *before* each pass); integrate into an existing weekly
workflow rather than launching a standalone app.
**Open question:** Is auto-ingestion of as-applied data technically feasible across the
equipment/software stack farmers actually run?
