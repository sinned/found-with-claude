# Claude for Founders — Positioning

How to think about and describe this skill pack to founders, investors, and the
Anthropic team.

---

## The thesis in one sentence

Founders do not just write code. Founders build companies. Claude should help with
the whole job.

---

## What the current narrative misses

Most "AI for founders" content — blog posts, demos, launch videos — shows founders
using AI to write code. The implicit message: AI makes you faster at building.

This is true. It is also insufficient.

The most common causes of startup failure:
- Building something nobody wants (wrong problem or wrong customer)
- Running out of money before finding PMF (misread the signals)
- Hiring the wrong first person (no framework for the decision)
- Pitching the wrong story to investors (narrative not built from evidence)
- Not knowing when to pivot vs. persist (weak weekly review discipline)

None of these are coding problems. All of them are problems that structured thinking
and fast synthesis can help with. That is where this skill pack lives.

---

## What this is NOT

**Not a chatbot for founders.** A chatbot conversation about your problem is
useful once. A structured skill with defined steps, defined output, and defined
success criteria is repeatable. Repeatability is what makes it a workflow tool
rather than a smart advisor.

**Not DevRel content.** The existing "AI for founders" content is mostly DevRel:
demos designed to show what Claude can do, not tools designed to be used. This is
a usable artifact, not a demo.

**Not a replacement for doing the work.** `/customer-discovery-synthesizer` does
not replace talking to customers. It synthesizes what you learned from talking to
customers. The skill cannot work without the founder doing the interviews first.

---

## What this IS

**A workflow accelerator for the full founder job.** Every skill in this pack
compresses a specific phase of the founder journey from hours to minutes. The
bottleneck becomes judgment, not synthesis.

**A thinking structure for decisions under uncertainty.** The opportunity map forces
explicit scoring. The product brief forces naming the riskiest assumption. The PMF
review forces a clear verdict. These structures work because they prevent the most
common failure mode: avoiding the uncomfortable question.

**A reusable skill pack that a founder can carry through their whole journey.**
Clone it once. Use it from week 1 to Demo Day. The skills compound — the opportunity
memo feeds the customer synthesis, which feeds the product brief, which feeds the
prototype scope.

---

## Who this is for

**Primary:** First-time founders at the idea stage or early-company stage (pre-Series A)

**Secondary:** Second-time founders who want to move faster through the early phases

**Not for:** Late-stage founders (different problems), non-founder operators (different context),
investors (different workflow — see a future `investing-with-claude` pack)

---

## How to describe it

**For a founder:** "It's a Claude Skills pack that helps you with every phase of
building a company — finding a problem, synthesizing customer interviews, writing
your product brief, scoping the prototype, finding first users, reading PMF signals,
and preparing for Demo Day. Clone it, open it in Claude Code, and the skills are
right there."

**For an investor:** "We're building the next stage of 'Claude for professionals'
tooling — not AI assistants, but structured workflow tools. This is Claude for
company-building, the same way prototype-with-claude is Claude for AI feature prototyping."

**For the Anthropic team:** "This shows that Claude Code skills can encode expertise
in any professional domain, not just software engineering. The pattern generalizes —
Claude for founders, Claude for investors, Claude for researchers, Claude for lawyers.
Each one is a self-contained skill pack for the full professional context."

---

## The broader pattern

```
prototype-with-claude
→ Claude for AI feature prototyping
→ Pattern: write behavior → eval → improve → ship

founding-with-claude
→ Claude for company building
→ Pattern: problem → customers → strategy → build → users → PMF → Demo Day

(future) investing-with-claude
→ Claude for sourcing, diligence, and portfolio ops

(future) research-with-claude
→ Claude for academic research workflows
```

Each repo is:
1. Self-contained (clone and use)
2. Domain-specific (built for one professional context)
3. Structured (skills with defined steps and outputs, not open-ended chats)
4. Grounded in real workflows (not hypothetical "you could also...")

The hypothesis: the most valuable use of Claude Code is not general-purpose AI
assistance — it is domain-specific workflow acceleration, where the domain knowledge
is encoded in skills and the founder/professional brings the raw material.
