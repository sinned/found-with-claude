# Investor Memo Template

Use this template structure when running /fundraising-narrative.
Fill in what you know; leave blanks for what you will develop with Claude.

---

# [Company Name] — Investor Memo
*[Stage: Pre-seed / Seed / Series A] | [Date]*

---

## The 30-second version
[3-4 sentences: problem, insight, what we built, traction.
Practice saying this out loud. If it takes more than 45 seconds, cut.]

---

## The problem

[Who suffers and how much. Specific person, specific situation, specific cost.
Not market size — the actual experience of the problem.]

---

## Why now

[What changed that makes this possible or necessary today.
Technology? Regulation? Behavior? If the answer is "nothing changed," think harder.]

---

## Our insight

[The non-obvious thing you learned from talking to customers.
What do you know that most people building in this space do not?]

---

## What we built

[The product, in plain language. What it does. Why this approach vs. others.
No architecture diagrams. No technical acronyms unless the audience is technical.]

---

## Traction

[Fill in what you have — be honest, no spin]

| Metric | Current | Growth rate |
|--------|---------|-------------|
| Active users / customers | | |
| Revenue (MRR / ARR) | | |
| Retention (week 2 or month 2) | | |
| Referrals (unprompted) | | |
| Key customers / pilots | | |

---

## Why us

[Why this team, specifically. What have you done — not what credentials you have —
that demonstrates you can build and sell this.
Founder-market fit: what do you know or see that others don't?]

---

## The ask

**Raising:** $[X]

**Use of funds:**
- [Specific: engineering headcount, integration development, etc.]
- [Not: "grow the team"]

**Milestone at the end of this round:**
[What you will prove with this money that makes the next round obvious.
Not "grow to $1M ARR" — "demonstrate that X by Y with Z evidence."]

---

## Run /fundraising-narrative with this context

Fill in what you know, then run `/fundraising-narrative`. Claude will complete the
narrative, write the objection map, and identify the most important question
you least want to be asked.
